# CLAUDE.md — Backend Service

> Read the root [CLAUDE.md](../../CLAUDE.md) first. This file adds backend-specific rules.

## Responsibilities

- Public REST API (`/api/v1/*`) and WebSocket endpoints
- AuthN/AuthZ (JWT + RBAC)
- Persistence (Postgres) and cache/pubsub (Redis)
- Orchestration of the AI engine, recording service, notification service
- Issuing signed playback URLs for recordings/snapshots

This service is **stateful only via Postgres + Redis** — never local files.
The recording service writes media; the backend only references it.

## Layout

```
app/
├── main.py             # FastAPI factory, lifespan, CORS, /metrics, /healthz
├── core/               # config, logging, security, exceptions
├── db/
│   ├── base.py         # DeclarativeBase + mixins
│   ├── session.py      # async engine + get_session
│   └── repositories/   # all SQL access lives here
├── models/             # SQLAlchemy ORM
├── schemas/            # Pydantic v2 (request/response)
├── services/           # business logic (auth, camera, storage, event-bus)
├── api/v1/
│   ├── deps.py         # FastAPI Depends() factories
│   ├── router.py       # aggregate router
│   └── endpoints/      # one module per resource
└── workers/            # Celery tasks (retention, archival, health)
```

## Layering Rules (enforced by review)

```
endpoints  ──calls──▶  services  ──calls──▶  repositories  ──uses──▶  models
   ▲                       │
   │                       └──may call other services
   └── only HTTP/WS concerns; no business logic, no SQL
```

- **Endpoints** parse input, call a service, shape output. Nothing else.
- **Services** are pure async functions on injected repos. No HTTP calls here
  unless to internal services (AI engine, notification service).
- **Repositories** are the only place `session.execute(...)` is written.
- **Models** never appear in API responses — convert to schemas.

## Database / Migrations

- Use **Alembic** for every schema change. Never `Base.metadata.create_all` in prod.
- Generate: `alembic revision --autogenerate -m "add foo"`
- Apply: `alembic upgrade head`
- All UUIDs are server-generated by the model default. Never accept client IDs.
- Always add an index for any column you `.where()` or `.order_by()` frequently.

## Auth

- Access tokens: 60 min default. Refresh tokens: 14 days default.
- Use `Depends(require_role(UserRole.ADMIN))` to guard mutating endpoints.
- WebSocket auth is via `?token=…` query param (no `Authorization` header in
  browser WS). Validate in the handler before `accept()`.

## Storage

- Always go through `app.services.storage.get_storage()` — never write to disk
  directly.
- `LocalStorage` returns a backend-served URL; `S3Storage` returns a presigned URL.
- Recording keys come from `make_recording_key(camera_id, started_at)` so they
  sort lexicographically by time and partition cleanly per-camera.

## Common Pitfalls

- `await session.commit()` is handled by the `get_session` dependency. Don't
  commit inside services.
- Pydantic v2: use `model_dump(exclude_unset=True)` for PATCH endpoints, not
  `dict(exclude_unset=True)`.
- Don't forget to `from app import models` in `alembic/env.py` so autogenerate
  sees them.
